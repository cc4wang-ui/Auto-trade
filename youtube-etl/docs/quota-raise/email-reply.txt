   (attachment: Planned_dashboard_layout.pdf)

The production data is presented to ~10 internal employees via Google Connected Sheets, sourced from BigQuery. The workbook contains five tabs covering the full range of analytics fetched:

  Tab 1 — Overview
    All 50 managed channels ranked by 7-day, 30-day, and 90-day
    views, likes, comments, and estimated revenue. Used by
    leadership for portfolio-level performance review.

  Tab 2 — Per-Manager
    Nine talent-management groups (Manzoku, Kamata, Heimao, Nagai,
    Takaichi, Shiratori, Katabami, Franky, Official) with aggregated
    metrics and per-talent comparison within each group. Used by
    each manager for team review.

  Tab 3 — Per-Talent (drill-down)
    Single-channel 90-day daily time series of:
      - views, uniqueViewers
      - estimatedMinutesWatched, averageViewDuration
      - estimatedRevenue, estimatedAdRevenue, cpm
      - subscribersGained, subscribersLost
      - top videos ranked by views and by comment velocity

  Tab 4 — Live Watch
    Channels currently broadcasting live, with concurrent viewer
    counts updated every 5 minutes via the videos.list endpoint
    (parts liveStreamingDetails + statistics). Historical concurrent
    peaks are captured during the broadcast and persisted to
    BigQuery, since they are not retrievable post-broadcast.

  Tab 5 — Tag Insights
    Content category distribution per talent (e.g. gaming, music,
    talk-show, collab) used for content-mix planning. Tags are
    derived internally from video title + description by an NLP step
    and stored alongside the metric data in BigQuery; no external
    classifier service is involved.

Refresh schedule : daily 09:00 JST via Connected Sheets scheduled
                   refresh, plus on-demand refresh by team members.
Access control   : view-only, restricted to ~10 internal production-
                   team members via Google Workspace SSO. The
                   workbook is not externally shared, not embedded
                   on any public site, and not exported to third
                   parties.

4. DATA FIELDS AND API METHODS
────────────────────────────────────────────────────────────

  YouTube Data API v3 endpoints used:
    - videos.list         (parts: statistics, snippet,
                           contentDetails, liveStreamingDetails)
    - commentThreads.list (top-level threads only; replies are not
                           fetched)
    - search.list         (recent uploads, hourly)
    - channels.list       (initial channel metadata, infrequent)

  YouTube Analytics API v2 — channel-level daily report metrics:
    - views                    - uniqueViewers
    - estimatedMinutesWatched  - averageViewDuration
    - estimatedRevenue         - estimatedAdRevenue
    - cpm                      - subscribersGained
    - subscribersLost          - likes
    - shares                   - comments

  Each Analytics report is one HTTP call per (channel, date). To
  account for YouTube's documented multi-day Analytics number
  finalisation, the pipeline re-fetches the past 7 days each run
  and upserts on (channel_id, report_date), so the stored numbers
  always reflect the latest YouTube-published values.
